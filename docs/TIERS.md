# AttentionOS 产品三档 & 数据联动设计 (draft v1)

## 三档定位

沿用大家熟悉的模型命名阶梯（这本身就是品牌梗的一部分：
*we benchmark models — now benchmark yourself*）：

| 档位 | 形态 | 数据深度 | 角色 |
|---|---|---|---|
| **AttentionOS mini** | 浏览器 60s 体检，零安装 | 快照（一次测试） | 获客 · 传播 |
| **AttentionOS** | 桌宠 + 本地采集器 + 每日结算卡 | 每日状态流 | 留存 · 习惯 |
| **AttentionOS Pro** | Dashboard + 进程归因 + 周报 + MCP server + Focus Shield | 纵向分析 | 深度 · 干预 |

原则：三档全部开源免费。"Pro" 表达的是**深度**而不是价格——它是同一个桌面
App 里的完整观测模块，不是付费墙。将来若商业化，边界放在可选的云同步/团队
功能上，个人 Pro 永远免费（隐私是本品类唯一护城河，收费采集数据会毁掉它）。

升级动线（每一档的结尾就是下一档的入口）：

```
mini 结果页  →  「把 [堡垒] 领养到桌面」   (存档码 / deep link)
桌宠结算卡  →  「查看完整报告」            (打开 Pro 模块)
Pro 周报    →  「分享战绩卡」              (回流 mini 传播)
```

## 数据联动：Attention Profile v1

测试结果是一个可携带的、版本化的本地档案。**没有账号、没有服务器**。

```json
{
  "v": 1,
  "ts": "2026-07-03T14:30:00+08:00",
  "type": "fortress",
  "scores": { "alerting": 78, "orienting": 64, "executive": 88 },
  "raw": { "s1rt": 287, "s2rt": 1340, "s3cost": 62, "s3acc": 100 },
  "lang": "zh"
}
```

四种传递通道，按实现顺序：

1. **存档码（save code）** — `attn1.<base64url(json)>`，展示态缩写为
   `FRT·78·64·88`。结果页一键复制，桌面端粘贴导入。像素/掌机风格里它就叫
   "存档密码"——上世纪游戏机没有存档芯片时代的原生梗，和产品气质完全一致。
2. **Deep link** — 桌面端注册 `attentionos://adopt?p=attn1.…` 协议；
   网页结果页的「领养」按钮直接唤起已安装的桌面端，未安装则跳转下载页。
3. **QR 码** — 分享卡角落嵌入 deep link 的 QR：手机测完 → 电脑扫码领养，
   顺便让每张社交媒体上的卡都自带安装入口。
4. **MCP server（Pro）** — 桌面端暴露 `attention://profile`（静态画像）与
   `attention://state`（实时：专注中/可打扰/恢复中），任何 Agent 或第三方工具
   按标准协议读取。这是"把数据传给其他工具"的正式答案：不做私有 API，
   直接站在 MCP 生态上。

## 联动语义（数据如何被下游消费）

- **type → 物种**：测试类型决定宠物物种（六个 16×16 精灵已就位）。
- **scores → 初始属性**：三网分数即宠物的三条属性条。
- **weakest → 训练目标**：最弱网络成为宠物的"每日任务"方向
  （执行弱 → 宠物发起 Focus Shield 挑战；警觉弱 → 提醒 90 分钟节律）。
- **raw → 基线**：桌面端把测试基线和日常采集数据放在同一张 SQLite 表，
  Pro 的周报可以回答"你的冲突代价比上周降了 14ms"。

宠物与 dashboard 是同一份本地 SQLite 的两个视图：宠物是**常驻的低分辨率
视图**（状态、情绪、进化），dashboard 是**按需的高分辨率视图**（时间线、
归因、趋势）。数据只写一次。

## 隐私边界（不可妥协）

采集数据永不出机器；存档码/QR 只含测试快照（用户主动生成）；
MCP 只在本机回环；导出永远是用户主动的 JSON/CSV。
